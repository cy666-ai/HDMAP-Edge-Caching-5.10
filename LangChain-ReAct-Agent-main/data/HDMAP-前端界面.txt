HLR-Cache 面向自动驾驶的分层高精地图边缘缓存系统 — 前端界面说明

前端基于 Vue 3 + Vite + Element Plus + ECharts + Leaflet 构建，采用单页面布局。
访问地址为 http://localhost:5173。

============================================================================
页面整体布局 (Dashboard.vue)
============================================================================

页面分为五大区域：
1. 标题栏（顶部白色，56px高）：显示系统名称"面向自动驾驶的分层高精地图边缘缓存系统"，右侧有"适配视图"按钮（点击后自动缩放地图使所有车辆可见）
2. 左侧地图区（占剩余宽度）：Leaflet 高德地图，显示车辆轨迹线、车辆 SVG 图标、RSU 标记及覆盖圈
3. 中间拖拽手柄（6px宽）：可左右拖拽调整右侧面板宽度（260px-600px，默认320px）
4. 右侧面板区：从上到下依次为模拟控制、路线管理、标签切换栏（实时监控 / 统计分析 / 对比分析）、标签对应的内容面板
5. 左下角浮动按钮：HLR-Cache助手 聊天入口

标签切换栏位于右侧面板内（ControlPanel 和 RouteManager 下方），共三个标签：
- "实时监控"（Monitor 图标）：显示 DataDisplay 实时数据面板
- "统计分析"（DataAnalysis 图标）：显示 StatisticsPanel 六张统计图表
- "对比分析"（TrendCharts 图标）：显示 ComparisonPanel 五种算法对比

当前选中的标签为蓝色填充，其余为灰色文字。

============================================================================
左下角 FAB 按钮 — "HLR-Cache助手" (ChatWidget.vue)
============================================================================

位置：页面左下角固定定位，蓝色圆形大按钮，带 ChatDotRound 对话气泡图标，显示"HLR-Cache助手"文字。
作用：点击后从左侧滑出 AI 智能聊天抽屉（el-drawer，宽度默认420px，可拖拽调整 320-900px）。

抽屉内功能：
- 标题栏：左侧显示"HLR-Cache助手"或"📋 历史记录"，右侧有"新对话"按钮（Plus 图标）和"历史"/"← 返回"切换按钮
- 历史面板：展示所有历史对话列表，每条显示标题、消息数和相对时间（刚刚/X分钟前/X小时前），可点击切换对话，可删除（红色 Delete 图标）
- 聊天面板：
  - 空状态：显示"您好！我是 HLR-Cache助手"及提示文字
  - 用户消息：右对齐蓝色气泡
  - AI 消息：左对齐灰色气泡，流式输出时显示闪烁光标
  - 输入区：底部输入框（placeholder "输入问题，按 Enter 发送..."）+ 发送按钮（Promotion 图标）
  - 流式生成时发送按钮变为红色停止按钮（Close 图标），点击可中止生成
- 状态指示：输入框下方显示 AI 助手连接状态——"AI 助手已连接"（绿色 ✓）/"AI 助手检测中..."（橙色加载中）/"AI 助手离线"（红色 ✗）
- 数据持久化：所有对话自动保存到浏览器 localStorage，刷新不丢失

============================================================================
模拟控制面板 (ControlPanel.vue)
============================================================================

标题：模拟控制，右侧显示当前状态标签：
- "已就绪"（灰色，info 类型）：模拟未启动
- "行驶中"（绿色，success 类型）：模拟运行中
- "已暂停"（橙色，warning 类型）：模拟已暂停

按钮（三个等宽水平排列）：
- 开始模拟（蓝色 primary，▶ VideoPlay 图标）：idle 状态启动模拟，paused 状态恢复模拟。运行中禁用。
- 暂停（橙色 warning，⏸ VideoPause 图标）：暂停模拟，仅运行中可点击。
- 重置（红色 danger，↻ Refresh 图标）：重置模拟回到初始状态，idle 状态禁用。

时间速率滑块：1x 到 10x，默认 5x。值越大车辆移动越快。滑条上标记 1、5、10 三档，显示当前值为"Nx"。步长 1。

路线车辆配置（可展开/收起，标题栏显示总车辆数"N 辆"和展开/收起指示）：
- 每条路线显示：路线名称（左侧，最长160px截断）、车辆数量和速度摘要（"N 辆 · N km/h"）
- 车辆数滑块：每条路线 1-10 辆，默认 5 辆，步长 1
- 速度滑块（绿色）：0-90 km/h，步长 5，默认 35 km/h

实时数据面板 (DataDisplay.vue) — 实时监控标签页内容
============================================================================

标题：实时数据，右侧显示 WebSocket 连接状态（"已连接"绿/"未连接"红）。

数据网格（2列）：
- 行驶时间：格式 HH:MM:SS（等宽字体）
- 车辆数量：当前活跃车辆总数（等宽字体）

选中车辆详情（点击地图上车辆图标后显示；未选中时显示"点击地图上的车辆查看详情"灰色提示文字）：
- 车辆ID（#N 格式）、行驶速度（km/h，绿色高亮18px）、纬度/经度（6位小数，等宽字体）、行驶方向（度）、轨迹点数
- 瓦片收集进度：已收集数/总请求数，带彩色进度条（≤20%红→≤50%橙→≤100%蓝→100%绿，stroke宽度14）
- 查看/收起瓦片列表按钮：展开后显示已收集的瓦片 ID（蓝色小标签）
- 沿途 RSU（实时）：该车辆前方将要经过的所有 RSU 编号（路线颜色标签），含总数"共 N 个 RSU"

底部嵌入缓存命中率组件（RSUHitRate）。

地图视图 (MapView.vue) — 左侧地图区域
============================================================================

地图引擎：Leaflet + 高德地图瓦片（webrd01.is.autonavi.com，中文标注，style 8）。
默认中心：[32.059, 118.769]（南京鼓楼区），默认缩放级别 15，缩放范围 3-18。
左上角有 +/- 缩放控件。

地图元素：
- RSU 标记：圆形实心点（半径8px，白色边框2px），颜色按所属路线区分（12色调色板），填充透明度根据命中率动态调整（命中率越高越不透明）。
- RSU 覆盖圈：每个 RSU 周围 250m 半径的半透明圆圈（fillOpacity 0.08，opacity 0.3），颜色与 RSU 所属路线一致。注意：此 250m 为地图可视范围圈，实际通信范围为 300m。
- RSU 弹窗（点击标记弹出）：显示"RSU #ID"、RSU 名称、所属路线、命中率%（仅更新后显示）、缓存瓦片数量及瓦片 ID 列表（蓝色小标签），无缓存时显示"暂无缓存"。
- 车辆图标：自定义 SVG 小车图标（32×32px），车身矩形（圆角3px）+ 挡风玻璃 + 两黑色车轮 + 白色车窗，颜色按所属路线区分。车头方向随行驶方向旋转。
- 车辆弹窗（点击图标弹出）：显示"车辆 #ID"、所属路线、将要途经的 RSU 数量和编号列表（路线颜色标签），无数据时显示"暂无数据"。
- 车辆轨迹线：每辆车行驶过的路径（折线，路线颜色，线宽3，透明度0.7）。

地图交互：
- 点击车辆图标 → 选中车辆，地图飞行到该位置并打开弹窗，右侧实时数据面板同步显示该车详情。
- 点击 RSU 标记 → 打开弹窗显示 RSU 详情。
- 地图取点模式（添加路线时触发）：地图顶部中央显示浮动提示条"请在地图上点击选择起点/终点"+ 取消按钮，光标变为十字准星，点击地图任意位置捕获经纬度坐标。

"适配视图"按钮（标题栏右侧，Aim 准星图标）：自动调整地图视野使所有车辆可见（maxZoom 18，50px 内边距）。

缓存命中率面板 (RSUHitRate.vue) — 嵌入实时数据面板底部
============================================================================

标题：缓存命中率（MWC），右侧显示 MWC 算法状态标签：
- "已就绪"（绿色 success）：算法就绪
- "MWC计算中"（橙色 warning）：算法正在执行
- "算法错误"（红色 danger）：算法执行出错
- "未计算"（灰色 info）：尚未执行过算法

总命中率：大号百分比数字（32px 加粗等宽字体）+ 进度条（stroke 宽度16）。颜色规则：>70%绿，40%-70%橙，≤40%红。未计算时灰色。

各路线 RSU 详情（按路线分组）：
- 路线名称（带路线颜色圆点）+ "N 辆车 · N 个RSU" 摘要
- 命中率%（保留一位小数）+ 命中块数（"N 块"）
- 进度条（stroke 宽度12，颜色同命中率规则）

底部操作行：
- 重新计算 按钮（蓝色 primary 圆形）：手动触发 MWC 算法重新优化缓存。计算中显示"MWC计算中..."并禁用，计算完成后自动恢复。
- 上次计算时间戳（"上次: HH:MM:SS"格式）。
- 当前时间片编号（"时间片 #N"，右下角浅灰色）。

统计分析面板 (StatisticsPanel.vue) — 统计分析标签页内容
============================================================================

标题：统计分析，右侧显示当前 tick 编号标签。

未收到仿真数据时显示"等待仿真数据..."空状态（DataAnalysis 灰色图标 + 文字）。

共 6 张 ECharts 图表（数据就绪后显示）：

1. 最大净效用 (MaxNetUtility)：横向柱状图，Y轴为路线名（截断6字符），X轴为净效用值，按路线着色。tooltip 显示路线ID、路线名、MaxNetUtility(Final)值、tick编号。

2. 缓存利用率：竖向柱状图，X轴为路线名（截断+旋转25度），Y轴为利用率百分比。每根柱子着色：≥80%绿，≥60%橙，<60%红。柱上显示精确百分比。

3. RSU 负载均衡度：
   - 仪表盘（120×120px）：4段颜色分区——[0-30%)红、[30-60%)橙、[60-80%)蓝、[80-100%]绿，起始角210°，结束角-30°，数值动画过渡
   - 右侧均衡度得分（大号百分比数字，颜色对应仪表盘区间）+ 文字描述（≥80%"负载分布非常均衡"，≥60%"负载分布较为均衡"，≥30%"存在一定不均衡"，<30%"负载分布差异较大"）
   - 下方负载分布柱状图：各路线平均访问车辆数（柱状）+ 各 RSU 实际车辆数（散点叠加）+ 误差范围线（min-max），Y轴为"访问车辆数"

4. 累积命中率曲线：折线图，X轴为时间片（tick），Y轴为命中率%。包含"系统平均"粗深色线（线宽3，#303133）和各路线细色线（线宽1.5，透明度0.7，平滑曲线）。图例可滚动。

5. 路线长度 vs 命中率趋势：折线图，X轴为 RSU 序号（从1开始），Y轴为命中率%。每个数据点代表该路线在对应 RSU 位置的命中率。数据优先使用实时车辆收集数据（realtimeRsuHitRates），回退使用算法模型数据（chrRsu）。圆点标记（大小5）。

6. 车辆密度 vs 命中率：散点图，X轴为"车辆密度 (辆/RSU)"，Y轴为命中率%。每个气泡代表一条路线，大小与车辆数成正比（18-36px），按路线着色。tooltip 显示路线名、密度、命中率、RSU数、车辆数。

每张图表右上角有下载按钮（Download 图标），点击导出为 PNG 图片（2倍像素密度，白色背景）。
图表随右侧面板宽度变化自动 resize（50ms 防抖）。
所有 tooltip 使用 appendToBody 模式，防止在侧边栏内被裁剪。

路线管理面板 (RouteManager.vue) — 右侧面板
============================================================================

标题：路线管理，右侧显示当前路线数量徽章（"N 条"）+ 展开/收起指示。

路线列表（面板展开时显示）：
- 每条路线显示：路线颜色圆点（8px）+ 路线名称（加粗13px）+ "起点 → 终点"（11px灰色）
- 右侧删除按钮（红色圆形 Delete 图标，size=small）
- 删除限制：模拟运行中禁用；仅剩 1 条路线时禁用
- 点击删除弹出 confirm 确认框："确定要删除路线「名称」吗？相关的 RSU、车辆和缓存决策将被移除。"

添加新路线（可展开/收起，标题栏可点击）：
表单字段（全部为 el-input size=small）：
- 路线名称：输入框，placeholder "路线名称（如：鼓楼→新街口）"
- 起点站名：输入框，placeholder "起点站名"
- 终点站名：输入框，placeholder "终点站名"
- 起点坐标：两列布局，左侧"起点坐标"标签，中间"地图取点"按钮（MapLocation 图标），右侧显示已选坐标值（蓝色等宽字体，6位小数）。点击"地图取点"后按钮变为橙色 warning 样式并显示"地图取点中..."，同时地图进入取点模式。
- 终点坐标：同上
- 获取高德路径 按钮：通过高德驾车路径规划 API 获取真实道路路径点。需先填好起终点坐标。获取中显示"获取中..."加载状态，成功显示"已获取 N 个路径点"（绿色），失败显示"获取失败: 原因"（红色）。
- 添加路线 按钮（蓝色 primary）：提交新路线到后端。需所有字段填写完整（名称、起点站名、终点站名、起点经纬度、终点经纬度）。提交中显示"添加中..."加载状态。成功后自动清空表单、收起添加表单。
- 【重要】添加路线时没有"RSU数量"输入框，用户不可设置、预估或指定 RSU 数量。RSU 由算法根据路线长度和 500 米间距自动部署，用户只能在地图上查看已部署的 RSU 标记点（各路线颜色不同）。

对比分析面板 (ComparisonPanel.vue) — 对比分析标签页内容
============================================================================

标题：对比分析，右侧显示分析时间戳（有数据时）。

运行对比 按钮（蓝色 primary，TrendCharts 图标）：触发 5 种算法（MWC / MPC / MAP / TRWC / MAMAB）在同条件下运行对比。分析中显示"分析中..."加载状态并禁用按钮。失败时显示红色错误信息。

5 种算法图例说明：
- MWC（蓝色 #5470C6）：最大权重闭包 — 本系统核心缓存优化算法
- MPC（绿色 #91CC75）：最流行缓存 — 按全局请求频率缓存
- MAP（黄色 #FAC858）：移动感知概率缓存 — 基于车辆移动概率
- TRWC（红色 #EE6666）：轨迹中继缓存 — 车辆间中继共享
- MAMAB（青色 #73C0DE）：多臂老虎机在线学习 — 在线自适应选择

3 张对比图表（每张右上角有 PNG 下载按钮）：

1. 命中率对比：分组柱状图，X轴为路线，Y轴为命中率%（0-100%）。每组 5 根柱子（5 种算法），柱上显示百分比标签。副标题显示当前车速。

2. 计算耗时对比：分组柱状图，X轴为路线，Y轴为耗时（毫秒）。每组 5 根柱子，柱上显示毫秒数标签。

3. 绝对命中/请求数对比：分组柱状图，X轴为路线，Y轴为数量。每组两系列（命中=实色柱带标签，请求=半透明柱 opacity 0.35）。5 种算法 × 2 系列 = 每组 10 根柱子。

详细数据表格（带条纹边框，size=small）：
- 列：路线（固定列，min-width 110）、RSU 数（宽55居中）、MWC、MPC、MAP、TRWC、MAMAB
- 每个算法单元格显示两行：大号加粗命中率%（算法颜色）+ 小号灰色耗时 ms
