HDMAP 分层高精地图边缘缓存可视化系统 — 前端界面说明

前端基于 Vue 3 + Vite + Element Plus + ECharts + Leaflet 构建，采用单页面布局。
访问地址为 http://localhost:5173。

============================================================================
页面整体布局 (Dashboard.vue)
============================================================================

页面分为四大区域：
1. 左侧边栏：可切换"实时监控"和"统计分析"两个标签页
2. 中央地图区：Leaflet 高德地图，显示路线轨迹、车辆图标、RSU 标记
3. 右侧面板区：模拟控制、路线管理、AI 聊天浮窗
4. 左下角浮动按钮：HLR-Cache助手 聊天入口

============================================================================
左下角 FAB 按钮 — "HLR-Cache助手"
============================================================================

位置：页面左下角，蓝色圆角按钮，带对话气泡图标，显示"HLR-Cache助手"文字。
作用：点击后从左侧滑出 AI 智能聊天抽屉，可以询问系统相关问题。
状态指示：输入框下方显示连接状态——"已连接"为绿色，"检测中"为橙色加载中，"离线"为红色。

============================================================================
模拟控制面板 (ControlPanel.vue)
============================================================================

标题：模拟控制，右侧显示当前状态标签（"已就绪"灰/"行驶中"绿/"已暂停"橙）。

按钮：
- 开始模拟（蓝色，▶ 图标）：启动车辆模拟。未运行或已暂停时可点击，运行中禁用。
- 暂停（橙色，⏸ 图标）：暂停模拟，仅运行中可点击。
- 重置（红色，↻ 图标）：重置模拟回到初始状态，仅非 idle 状态可点击。

时间速率滑块：1x 到 10x。值越大车辆移动越快。默认为 1x。滑条上标记 1、5、10。

路线车辆配置（可展开/收起）：
- 每条路线显示路线名称、车辆数量、速度(km/h)
- 车辆数滑块：每条路线 1-10 辆车
- 速度滑块（绿色）：0-90 km/h，步长 5

============================================================================
实时数据面板 (DataDisplay.vue) — 实时监控标签页
============================================================================

标题：实时数据，右侧显示 WebSocket 连接状态（"已连接"绿/"未连接"红）。

数据网格：
- 行驶时间：格式 HH:MM:SS
- 车辆数量：当前活跃车辆总数

选中车辆详情（点击地图上车辆图标后显示）：
- 车辆ID、行驶速度(km/h)、纬度、经度、行驶方向(度)、轨迹点数
- 瓦片收集进度：已收集数/总请求数，带彩色进度条（红→橙→蓝→绿）
- 查看/收起瓦片列表：显示已收集的瓦片 ID 标签
- 沿途 RSU：该车辆前方将要经过的所有 RSU 编号（彩色标签）

底部嵌入缓存命中率组件。

============================================================================
地图视图 (MapView.vue) — 中央地图区域
============================================================================

地图引擎：Leaflet + 高德地图瓦片。

元素：
- RSU 标记（彩色圆点）：位置和覆盖半径圆圈（250m 半透明），点击弹出详情窗口显示 RSU 编号、所属路线、命中率、缓存瓦片列表。颜色对应路线。
- 车辆图标（小汽车 SVG）：车头方向即行驶方向，颜色对应路线（每条路线 5 辆车同色）。
- 轨迹线：车辆行驶过的路径（按路线着色）。

交互：
- 点击车辆 → 选中车辆，飞行到该位置，右侧显示该车详情。
- 缩放控件：左上角 +/- 按钮。
- 路线变更/重置时自动清除旧 RSU 标记和覆盖圈。

============================================================================
缓存命中率面板 (RSUHitRate.vue) — 嵌入实时数据面板底部
============================================================================

标题：缓存命中率（MWC），右侧显示 MWC 算法状态标签（"已就绪"绿/"MWC计算中"橙/"算法错误"红/"未计算"灰）。

总命中率：大号百分比数字 + 进度条。
各路线详情：每条路线的命中率百分比、命中块数、进度条。
底部操作：
- 重新计算 按钮：手动触发 MWC 算法重新优化缓存。计算中显示"MWC计算中..."并禁用。
- 上次计算时间戳。
- 当前时间片 tick 编号。

颜色规则：命中率 > 70% 绿色，40%-70% 橙色，< 40% 红色。

============================================================================
统计分析面板 (StatisticsPanel.vue) — 左侧统计分析标签页
============================================================================

标题：统计分析，右侧显示当前 tick 编号。

共 6 张 ECharts 图表：
1. 最大净效用 (MaxNetUtility)：横向柱状图，各路线 MWC 最终净效用值
2. 缓存利用率：竖向柱状图，各路线缓存使用百分比。≥80%绿，≥60%橙，<60%红
3. RSU 负载均衡度：仪表盘（0-100%）+ 柱状图（各路线平均负载 + 散点 + 误差范围）。均衡度 >=80%绿色，>=60%蓝色，>=30%橙色，否则红色
4. 累积命中率曲线：折线图，系统平均和各路线命中率随 tick 变化趋势
5. 路线长度 vs 命中率趋势：折线图，各路线随 RSU 序号增加的命中率变化
6. 车辆密度 vs 命中率：散点图，每个点代表一条路线，气泡大小对应车辆数。

每张图表右上角有下载按钮（↓ 图标），可导出为 PNG 图片。
图表 tooltip 使用 appendToBody 模式，防止在侧边栏内被裁剪。

============================================================================
路线管理面板 (RouteManager.vue) — 右侧面板
============================================================================

标题：路线管理，右侧显示当前路线数量和展开/收起。

路线列表：每条路线显示彩色圆点、路线名称、起点→终点，右侧有删除按钮（红色圆形垃圾桶图标）。仅剩余 1 条路线时或模拟运行中删除按钮禁用。

添加新路线（可展开）：
- 路线名称 输入框
- 起点站名 输入框
- 终点站名 输入框
- 起点坐标：地图取点按钮（点击后在地图上点选）+ 显示坐标值
- 终点坐标：同上
- 获取高德路径 按钮：通过高德驾车路径规划 API 获取真实道路路径点
- 添加路线 按钮：提交新路线到后端

============================================================================
对比分析面板 (ComparisonPanel.vue) — 右侧对比分析标签页
============================================================================

标题：对比分析，右侧显示分析时间戳。

运行对比 按钮：触发 5 种算法同条件运行对比，分析中显示"分析中..."。

5 种算法图例说明：
- MWC（蓝色）：最大权重闭包 — 本系统核心算法
- MPC（绿色）：最流行缓存 — 按全局请求频率缓存
- MAP（黄色）：移动感知概率缓存 — 基于车辆移动概率
- TRWC（红色）：轨迹中继缓存 — 车辆间中继共享
- MAMAB（青色）：多臂老虎机在线学习 — 在线自适应选择

3 张对比图表：
1. 命中率对比：分组柱状图，5 算法 × 6 路线
2. 计算耗时对比：分组柱状图，单位毫秒
3. 绝对命中/请求数对比：双系列柱状图（实心=命中，半透明=请求）

详细数据表格：每行一条路线，每列一种算法的命中率%和耗时ms。
