================================================================================
HDMAP 边缘缓存系统 — 路由与车辆配置
================================================================================

## 默认路线（6 条南京鼓楼区地铁路线）
1. 古平岗 → 新庄（路线ID: 1）：南京鼓楼区北部东西向路线，约 7km
2. 草场门 → 九华山（路线ID: 2）：南京鼓楼区中部东西向路线，约 6km
3. 龙江 → 鸡鸣寺（路线ID: 3）：南京鼓楼区西部到东部路线，约 6.5km
4. 鼓楼 → 玄武门（路线ID: 4）：南京鼓楼中心区短路线，约 4km
5. 新模范马路 → 岗子村（路线ID: 5）：南京鼓楼区北部路线，约 5km
6. 汉中门 → 明故宫（路线ID: 6）：南京鼓楼区南部东西向路线，约 5.5km

## 路线特性
- 所有路线基于高德 Amap Driving Route API v5 获取真实道路坐标
- 路线坐标序列包含数百个路径点（用于车辆平滑移动和 RSU 部署）
- 系统启动时强制使用高德 API 获取路径，不允许线性插值回退
- 路线数据持久化于 route_paths.json 缓存文件

## 车辆配置
- 默认车辆总数：30 辆（每条路线 5 辆）
- 车速范围：20-60 km/h（可由速度滑块调整，1-10 级）
- 默认速度：约 35 km/h
- 每 tick 的移动间隔取决于速度等级（1 = 1000ms, 10 = 100ms）

## 车辆模拟规则
- 每辆车沿路线的坐标序列逐个点移动
- 车辆到达路线终点后循环回到起点继续行驶
- 车辆始终沿道路行驶（使用高德真实路径坐标）
- 每 tick 计算车辆与各 RSU 的距离，判断是否进入覆盖范围

## 车辆瓦片请求
- 每辆车在行驶过程中持续请求地图瓦片
- 请求的瓦片由当前位置 + 预测路径决定
- 当车辆进入 RSU 300m 覆盖范围时，请求的瓦片与 RSU 缓存进行匹配
- 命中 = 请求瓦片在 RSU 缓存中，未命中 = 请求瓦片不在缓存中

## 路线管理
- 支持运行时动态添加/删除路线（通过 RouteManager 面板）
- 添加路线必须通过高德 API 获取真实路径坐标
- 至少保留 1 条路线（不允许全部删除）
- 路线变更后自动重新初始化模拟服务和 RSU 部署

## 高德 API 集成
- 使用 Amap Driving Route API v5
- 需要 AMAP_API_KEY 环境变量
- 路线获取失败时回滚操作，不允许无真实路径的路线存在
