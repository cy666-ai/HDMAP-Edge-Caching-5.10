================================================================================
HDMAP 边缘缓存系统 — 常见问题与故障排查
================================================================================

## 启动相关

### Q: 后端启动失败，报 "SQLite error"
A: 检查 simmap1.0/backend/data/ 目录是否有写入权限。SQLite 数据库文件会自动创建。可尝试删除旧的 .db 文件后重启。

### Q: 前端白屏或无法加载
A:
1. 确保后端已启动 (localhost:3000 /health 可访问)
2. 确保前端 Vite 开发服务器已启动 (localhost:5173)
3. 检查浏览器控制台是否有 CORS 错误
4. 确认 CORS 配置包含 http://localhost:5173 和 http://127.0.0.1:5173

### Q: 地图不显示
A:
1. 检查网络连接，Leaflet 瓦片从 CDN 加载
2. 高德地图瓦片可能需要 API Key
3. 检查 index.html 中 Leaflet CSS CDN 链接是否可访问

## 算法相关

### Q: Python 算法超时或失败
A:
1. 检查 Python 环境是否安装了所有依赖 (pip install -r requirements.txt)
2. 算法超时时间设置为 120 秒，复杂路线可能接近此限
3. 查看 Node.js 控制台 stderr 输出了解具体错误
4. 手动运行 python hm_export_cache_decision.py 测试

### Q: 命中率一直为 0
A:
1. 确保模拟正在运行（点击"开始"按钮）
2. 车辆需要进入 RSU 300m 范围才会统计命中率
3. 等待至少 5 个 tick 让算法首次执行
4. 检查 cache_decision.json 是否正常生成

### Q: RSU 数据不更新
A:
1. rsu:update 事件每 5 秒广播一次
2. 检查 Socket.IO 连接状态（前端 DataDisplay 面板显示连接状态）
3. 手动点击 RSUHitRate 面板的"重新计算"按钮触发算法

## 路线相关

### Q: 添加路线失败
A:
1. 需要高德 API Key (AMAP_API_KEY 环境变量)
2. 起终点坐标必须可路由（不能在海中或无路区域）
3. 检查高德 API 配额是否用尽
4. 路线名不能为空

### Q: 车辆不动
A:
1. 检查模拟是否处于"已暂停"状态，点击"继续"
2. 速度滑块不能为 0
3. 重置模拟后需要重新点击"开始"

## AI 助手相关

### Q: AI 聊天机器人无响应
A:
1. 确保 Python Agent 已启动 (python agent_server.py, 端口 8000)
2. 检查 DASHSCOPE_API_KEY 环境变量是否设置
3. 检查 Node.js 后端控制台是否有 chatService 连接错误
4. 检查 Chroma 向量数据库是否正确索引了知识库文档

### Q: AI 回答不准确
A:
1. 知识库文档可能需要更新或补充
2. 系统提示词可能需要调整
3. 工具返回的模拟数据为随机生成（在 stub 模式下），实际部署时连接后端可获取真实数据
