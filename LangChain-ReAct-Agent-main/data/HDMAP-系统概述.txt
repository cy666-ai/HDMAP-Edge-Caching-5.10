================================================================================
HDMAP 分层高精地图边缘缓存可视化系统 — 系统概述
================================================================================

## 项目定位
面向自动驾驶的分层高精地图边缘缓存可视化系统 (Layered HD-MAP Edge Caching Visualization System for Autonomous Driving)。系统模拟自动驾驶车辆在南京真实道路路线上行驶，每辆车经过一串路侧单元 (RSU)，RSU 负责缓存地图瓦片。Python MWC（最大权重闭包）算法周期性优化各 RSU 应缓存哪些瓦片以最大化预期命中率。Vue 3 前端通过 Socket.IO 实时可视化这一切。

## 技术架构
- 前端：Vue 3 + Vite + Element Plus 2.5 + ECharts 6 + Leaflet 1.9
- 后端：Node.js Express + Socket.IO + Sequelize/SQLite
- 算法：Python（LangChain/LangGraph + FastAPI + Chroma 向量数据库 + DashScope LLM）
- 地图服务：高德 (Amap) Driving Route API v5
- 数据库：SQLite（存储路线、车辆、轨迹数据）

## 系统组件
1. 后端服务层
   - server.js：启动入口，创建 HTTP Server + Socket.IO
   - app.js：Express 应用，CORS + JSON 中间件 + /health + /api/* 路由
   - routes/api.js：REST API（/api/roads, /api/vehicles, /api/rsu 等）
   - models/：Sequelize ORM 模型（Road, Vehicle, Trajectory）+ SQLite
   - socket/index.js：Socket.IO 事件总线（15+ 事件）
   - services/simulationService.js：车辆模拟引擎（30辆车，6条路线）
   - services/cachingService.js：RSU 缓存命中率引擎 + MWC 算法触发
   - services/serviceRegistry.js：跨模块服务实例共享

2. 前端 UI 层
   - Dashboard.vue：主布局编排器，Socket 监听 + 数据分发
   - MapView.vue：Leaflet 地图（车辆标记、RSU 覆盖圈、轨迹线）
   - ControlPanel.vue：模拟控制（开始/暂停/重置/速度滑块）
   - RouteManager.vue：路线管理（添加/删除，高德路径获取）
   - DataDisplay.vue：实时监控面板（车辆状态、瓦片收集进度）
   - RSUHitRate.vue：命中率展示（全系统 + 按路线）
   - StatisticsPanel.vue：6 个 ECharts 统计图表
   - ComparisonPanel.vue：5 算法对比分析

3. Python 算法层
   - hm_export_cache_decision.py：主入口，7 阶段流水线
   - probability_distribution.py：Poisson + Power-law 请求概率模型
   - hm_mwc_solver.py：networkx 最小割/最大流 MWC 求解
   - capacity_refinement_multi_rsu.py：多 RSU 容量优化 + 依赖修复
   - 4 个 benchmark_*.py：MPC/MAP/TRWC/MAMAB 基线对比

4. AI 助手层
   - agent_server.py：FastAPI SSE 流式聊天端点
   - agent/react_agent.py：LangGraph ReAct Agent
   - agent/tools/agent_tools.py：7 个工具（RAG/状态/路线/缓存）
   - rag/：Chroma 向量数据库 RAG 检索服务

## 数据流（每 Tick）
1. SimulationService 定时器触发 → 移动各车辆沿路线行进
2. broadcastData() → emit vehicle:update（所有车辆位置）
3. cachingService.onVehicleTick() → 跟踪车辆进入 RSU 覆盖范围
4. 每 5 tick → spawn Python 算法 → 重算最优缓存
5. CachingService.startBroadcast() → emit rsu:update（每 5 秒）

## Socket.IO 事件规范
采用 domain:action 命名：
- 模拟控制：simulation:start/pause/resume/reset
- 数据广播：vehicle:update, rsu:update, simulation:status
- 路线管理：route:getConfig/fetchAmap/add/delete + 对应的 Result 事件
- 算法：rsu:recalc, comparison:run/result
- 聊天（新增）：chat:send/chunk/stop/error
